Bem-vindo ao RODA!

Um repositório digital open-source projetado para a preservação

O RODA é uma solução de repositório digital que oferece funcionalidade a todas as principais unidades funcionais do modelo de referência OAIS. O RODA é capaz de ingerir, gerir e dar acesso aos diversos tipos de objetos digitais produzidos no âmbito das atividades de grandes empresas ou entidades públicas. O RODA é baseado em tecnologias open-source e é suportado por normas já existentes, tais como o Open Archival Information System (OAIS), Metadata Encoding and Transmission Standard (METS), Encoded Archival Description (EAD), Dublin Core (DC) e Metadados de preservação (PREMIS).

Conforma-se com normas abertas

O RODA segue normas abertas utilizando a EAD para metadados de descrição, PREMIS para metadados de preservação, METS para metadados estruturais e várias normas para metadados técnicos (por exemplo, NISO Z39.87 para imagens fixas digitais).

Independente do fornecedor

O RODA é 100% baseado em tecnologias open-source. Toda a infraestrutura necessária para suportar o RODA é independente do fornecedor. Isso significa que se podem usar as distribuições de hardware e Linux que melhor se adequam às suas necessidades institucionais.

Escalável

A natureza orientada para os serviços do RODA permite que o sistema seja altamente escalável, permitindo a distribuição da carga de processamento entre vários servidores. Além disso, o uso de componentes avançados de indexação permite que os serviços de descoberta do RODA sejam distribuídos por vários servidores num cluster para um desempenho ainda maior.

Ações de preservação incorporadas

As ações de preservação e gestão no interior do RODA são desempenhadas por um módulo de execução de tarefas. O módulo de execução de tarefas permite ao gestor do repositório executar tarefas de preservação num dado conjunto de dados. As ações de preservação incluem conversões de formatos, verificações de checksum, comunicação (por exemplo, para enviar automaticamente e-mails de aceitação/rejeição do SIP), verificações antivírus, etc.

Autenticidade

O RODA usa metadados de preservação (PREMIS) para criar uma cadeia de confiança entre todas as migrações de formatos e verificações de conteúdo. Os metadados de preservação, em conjunto com o estabelecimento da confiança do seu meio envolvente (ISO 16363), asseguram a fiabilidade do serviço e a autenticidade dos registos digitais fechados.

Suporte para vários formatos

O RODA é capaz de ingerir todos os tipos de conteúdo. Os componentes de ação de migração suportam a migração de documentos de texto, imagens raster, bases de dados relacionais, vídeo e áudio para formatos normalizados para preservação a longo prazo. Um mecanismo de plug-ins permite que o RODA suporte facilmente migrações de formatos adicionais.

Lida com a natureza de rápida mudança da tecnologia

O módulo de plug-ins e de execução de tarefas disponibiliza uma maneira simples de adicionar mais funcionalidade ao sistema (por exemplo, novas ações de preservação, alertas, ferramentas, etc.). Além disso, a arquitetura orientada para os serviços do RODA permite que os seus componentes sejam progressivamente atualizados, permitindo um uso heterogêneo de tecnologias a todos os níveis: hardware, SO e aplicações.

Controlo de acesso avançado

Os utilizadores têm de estar autenticados antes de acederem ao repositório. Todas as ações do utilizador são registadas para responsabilização futura. As permissões são granulares e podem ser definidas desde o nível do repositório até ao nível dos objetos de dados individuais.

Integração com sistemas de terceiros

O RODA expõe todas as suas funcionalidades através de uma API de serviços REST bem documentada. Bibliotecas Java convenientes encontram-se disponíveis no GitHub para permitir que os programadores interajam com o RODA através das suas Core APIs. Várias ferramentas existem para criar e manipular SIPs e submetê-los ao workflow de ingestão do RODA.

Workflow de ingestão avançado

O RODA suporta a ingestão de novos materiais digitais, bem como metadados associados em vários formatos distintos. São fornecidas ferramentas para permitir que os Produtores criem SIPs nos formatos suportados. O workflow de ingestão pode ser personalizado na interface do utilizador para lidar com políticas e dados especiais.

E muito mais...

O RODA é um sistema de repositório digital a longo prazo que implementa um workflow de ingestão que não só valida SIPs, mas também procura no seu conteúdo pela presença de vírus, realiza identificação de formatos, extrai metadados técnicos e migra formatos de ficheiro para substitutos mais "preserváveis".

O RODA também dá acesso a informações digitais de várias maneiras, tais como busca/navegação nos metadados disponíveis, assim como visualização online e download de originais, formatos de preservação e derivadas de disseminação.

As interfaces de administração permitem que os utilizadores do backoffice do repositório façam a gestão dos fonds/coleções e definam regras para as ações de preservação. Todas as interações entre os utilizadores (humanos e máquinas) e o repositório são registadas por razões de segurança e responsabilização.

O RODA garante que os dados ingeridos são autênticos através do registo dos metadados PREMIS em todas as ações executadas pelo repositório, da proveniência dos registos usando normas de metadados como EAD ou DC, e assegura a integridade e a disponibilidade através de uma monitorização frequente dos dados de modo a garantir que estes não foram adulterados.
